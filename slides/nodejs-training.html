Bila guna Node.js?

Sesuai — dan kurang sesuai.

✅ Sangat sesuai

REST / GraphQL API · aplikasi masa nyata (chat, notifikasi, WebSocket) · backend SPA · microservices · streaming · sistem banyak I/O (portal aduan, tempahan) · alat CLI & serverless.

⚠️ Kurang sesuai

Tugas berat CPU — pemprosesan imej/video, latihan AI/ML, kiraan saintifik besar. Satu utas utama boleh tersekat.
Penyelesaian: Worker Threads, job queue, atau bahasa lain.

Projek kursus (sistem aduan) = banyak I/O, tiada kiraan berat → padanan sempurna untuk Node.js.

Kenapa MongoDB?

SQL berbanding NoSQL (MongoDB).

AspekSQL (MySQL/PostgreSQL)NoSQL — MongoDB
Struktur dataJadual, baris, lajurDokumen JSON

MongoDB simpan dokumen JSON — sepadan dengan objek JavaScript. SQL & NoSQL ialah alat berbeza untuk masalah berbeza.

Bila guna MongoDB?

Sesuai — dan kurang sesuai.

✅ Sangat sesuai

Skema yang berubah-ubah · data JSON (REST API) · prototaip pantas · sistem kandungan (CMS) & katalog · analitik masa nyata & log · IoT · data bersarang/hierarki.

⚠️ Kurang sesuai

Data sangat relational dengan banyak JOIN kompleks & ACID ketat (cth: lejar kewangan) → SQL lebih sesuai.
(MongoDB moden kini menyokong transaksi multi-dokumen.)

Satu aduan = satu dokumen dengan medan pelbagai yang boleh berkembang → padanan baik untuk MongoDB.

Kenapa stack ini?

Kenapa Node.js + MongoDB?

Node.js

Satu bahasa (JavaScript) depan & belakang · I/O tak menyekat (banyak pengguna serentak) · pantas (V8) · ekosistem npm terbesar.

Sesuai: API, aplikasi masa nyata, sistem banyak I/O seperti portal aduan.

MongoDB

Simpan dokumen JSON (sepadan dengan objek JavaScript) · skema fleksibel · mudah skala mendatar.

Sesuai: data yang berubah, JSON, prototaip pantas, CRUD & CMS.

JSON hujung ke hujung: pelayar → Node.js → MongoDB tanpa "terjemahan" antara lapisan — pembangunan lebih pantas.

Langkah 2

Controller: Cipta & Baca.

// Baca + carian
exports.index = async (req, res) => {
  const { carian } = req.query;
  const pertanyaan = {};
  if (carian) {
    pertanyaan.$or = [
      { namaPengadu: { $regex: carian, $options: "i" } },
      // ...boleh tambah noAduan, premis
    ];
  }
  const senaraiAduan = await Aduan.find(pertanyaan).sort({ createdAt: -1 });
  res.render("aduan/index", { senaraiAduan, carian });
};
// Cipta aduan baru
exports.store = async (req, res) => {
  const aduan = await Aduan.create(req.body);
  res.redirect(`/aduan/${aduan.id}`);
};

Deployment di AWS

AWS — pilihan berskala besar.

Jalankan Node.js

Elastic Beanstalk — PaaS, paling mudah · App Runner — kontena terurus · Lightsail — VPS mudah · EC2 — kawalan penuh (+ PM2/Nginx) · Lambda + API Gateway — serverless · ECS / Fargate — Docker.

MongoDB di AWS

MongoDB Atlas (di AWS) — terurus, disyorkan · Amazon DocumentDB — serasi dengan MongoDB (API-compatible) · MongoDB di EC2 — pasang & urus sendiri.

AWS lebih berkuasa tetapi kompleks. Pemula: mula dengan Render + Atlas; masuk AWS melalui Elastic Beanstalk + Atlas / DocumentDB.
